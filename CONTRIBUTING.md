# Contributing

VibeSpec은 SDD(Spec-Driven Development) 기반 바이브코딩을 위한 플러그인입니다. 기여를 환영합니다.

## 기여 방법

- **버그 수정, 오타** — PR을 직접 열어주세요.
- **새 스킬, 에이전트, 큰 변경** — 이슈를 먼저 열어 접근 방법을 논의한 후 PR을 제출해주세요.

## 네이밍 컨벤션

| 유형 | 접두사 | 패턴 | 예시 |
|------|--------|------|------|
| **커맨드 스킬** | `vs-` | `vs-{동사}` | `vs-plan`, `vs-commit`, `vs-merge` |
| **도메인 스킬** | 없음 | `{주제}` | `sdd-methodology`, `tdd-principles` |
| **에이전트** | 없음 | `{역할}` | `spec-writer`, `tdd-implementer` |

- 커맨드 스킬은 사용자가 `/vs-plan`처럼 호출하는 워크플로우입니다
- 도메인 스킬은 Claude가 자동으로 참조하는 지식 베이스입니다
- `vs-` 접두사는 커맨드 스킬에만 사용합니다

## 스킬 작성 규칙

### 공통

- `skills/{name}/SKILL.md` 파일에 작성합니다
- 프론트매터에 `name`과 `description`이 필수입니다
- `name`은 디렉토리명과 반드시 일치해야 합니다

### 커맨드 스킬

- `**체크포인트**:` 마커로 사용자 확인 지점을 명시합니다
- 체크포인트 메시지에 구체적 선택지를 포함합니다
- `## 다음 단계` 섹션을 스킬 마지막에 배치합니다 (3-4개 추천)

### 도메인 스킬

- `Domain Context` → `Key Principles` → 상세 섹션 → `참고자료` 구조를 따릅니다
- 500단어 이상 2000단어 이하를 목표로 합니다

## 검증

PR 제출 전에 반드시 실행하세요:

```bash
npx tsx scripts/validate-plugin.ts
```
