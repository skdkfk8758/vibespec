# UX Defaults — 자동 진행 기본 원칙

VibeSpec 스킬의 대화형 체크포인트 동작을 정의하는 기준 문서. 모든 스킬 구현이 이 원칙을 따른다.

> **배경**: 스킬 기능이 추가될 때마다 `AskUserQuestion` 체크포인트가 누적되어 평균 세션에서 질문이 과다해짐 (플랜 `8uu3rflthWIg` 참조). 자동 진행을 기본값으로 만들어 사용자의 대화 부담을 줄인다.

## 핵심 원칙

### 1. Default ON — 자동 진행이 기본

- 사용자가 별도 플래그 없이 스킬을 호출하면 **자동 진행 모드**로 실행된다.
- 자동 모드에서는 각 체크포인트가 **기본값 옵션을 자동 선택**하고, 핵심 결정 지점만 질문한다.
- 이전에 각 질문의 "권장" 또는 첫 번째 선택지가 자동 선택된다 (스킬별 명시된 기본값 참조).

### 2. Opt-in to Interactive — `--interactive` 플래그

- 사용자가 `--interactive` 플래그를 전달하면 **기존 전체 대화형 체크포인트**가 활성화된다.
- 체이닝된 하위 스킬에도 플래그가 전파된다.
- 이 플래그는 "나는 지금 신중하게 판단하고 싶다"는 사용자의 명시적 선택이다.

```
# 자동 진행 (기본)
/vs-next

# 기존 방식 (모든 체크포인트 활성화)
/vs-next --interactive
```

### 3. Safety Guards Bypass — 자동 진행 예외 5종

아래 5개 카테고리는 `--interactive` 플래그 유무와 무관하게 **반드시 사용자 확인**을 받는다.

| 카테고리 | 설명 | 예시 |
|---|---|---|
| **파괴적 명령** | 되돌릴 수 없는 작업 | force push, 브랜치 삭제, force merge |
| **강제 완료 사유** | FAIL verdict를 override할 때 | verifier FAIL 상태에서 "강제 완료" 선택 시 사유 입력 |
| **머지 충돌** | 자동 해결 불가한 충돌 | add/add 충돌에서 파일 선택 |
| **백업 덮어쓰기** | 기존 파일을 덮어쓸 때 | DESIGN.md 재생성 시 기존 내용 덮어쓰기 |
| **Secrets 감지** | 커밋 시 민감 정보 의심 | .env, credentials.json 등 staged 시 경고 |

### 4. Skill Chaining — 중복 질문 스킵

한 스킬이 다른 스킬을 호출할 때(예: `vs-next` → `vs-commit`), 상위 스킬에서 이미 확인한 결정은 **재질문하지 않는다**.

- 구현 방법: 상위 스킬이 하위 스킬 호출 시 체이닝 컨텍스트(예: `--from-vs-next`, `--skip-phase7`)를 전달
- 예시: `/vs-next`의 완료 체크포인트에서 "커밋" 선택 시, `/vs-commit`의 Phase 7 "플랜 검증 실행?" 재질문을 자동 스킵

## 스킬별 기본값 테이블

### vs-next

| 체크포인트 | 자동 모드 기본값 | `--interactive` 모드 |
|---|---|---|
| stash 복원 (현재 브랜치 1개 매칭) | **자동 복원 + 알림** | 복원/건너뛰기/삭제 선택 |
| 워크트리 확인 | **경고 안내만** | 워크트리 생성/그대로 진행 |
| Complex 태스크 재추천 | 스킵 (trigger 미충족 시) | 워크트리 생성/그대로 진행 |
| 구현 방식 선택 | **TDD 자동 판별 테이블 적용** | TDD/직접/건너뛰기 |
| 마일스톤 QA 트리거 | **나중에** (첫 도달 시만 1회 알림) | QA 실행/나중에 |
| 완료 후 체크포인트 | **다음 태스크 자동** | 다음/배치/커밋/대시보드 |
| **FAIL 강제 완료 사유** | ✕ 예외 — 항상 질문 | 항상 질문 |

### vs-commit

| 체크포인트 | 자동 모드 기본값 | `--interactive` 모드 |
|---|---|---|
| Phase 0 simplify-loop | **건너뛰기** | simplify-loop/건너뛰기 |
| Phase 4 커밋 계획 | **자동 승인** (계획은 표시) | Y/수정/재분류 |
| staged 파일 처리 | **전체 통합 분석** | staged-only/전체/분리 |
| Phase 7 플랜 검증 | **스킵 (체이닝 시 항상)** | 플랜 검증/나중에 |
| **Secrets 감지** | ✕ 예외 — 항상 경고 | 항상 경고 |

### vs-plan-verify

| 체크포인트 | 자동 모드 기본값 | `--interactive` 모드 |
|---|---|---|
| 플랜 선택 (다중) | ✕ 예외 — 항상 질문 | 항상 질문 |
| 스켈레톤 WARN/FAIL | **무시** (리포트에만 기록) | 업데이트/무시 |
| PASS 후 체크포인트 | **안내만 표시** | 플랜 완료/머지/유지 |
| **FAIL 강제 완료 사유** | ✕ 예외 — 항상 질문 | 항상 질문 |

### vs-plan

| 체크포인트 | 자동 모드 기본값 | `--interactive` 모드 |
|---|---|---|
| 백로그 중복 감지 | **새로 플래닝** (알림 1줄) | 승격/새로 플래닝 |
| Step 6 통합 체크포인트 | ✕ 예외 — 항상 질문 (유일한 필수 확인) | 항상 질문 |
| Step 7a 디자인 점수 | 조건부 (D/F 등급만 질문) | 조건부 |
| Step 8 워크트리 추천 | **안내만** | 워크트리/현재/나중에 |

### vs-exec

| 체크포인트 | 자동 모드 기본값 | `--interactive` 모드 |
|---|---|---|
| careful 모드 제안 | **건너뛰기** (이미 on이면 스킵) | 활성화/건너뛰기 |
| Phase 2 플랜 리뷰 | **실행 시작** (리뷰 리포트는 표시) | 실행시작/스펙수정/취소 |
| Phase 4 종합 리포트 | **안내만** | 커밋/리뷰/대시보드 |

## 구현 지침 (스킬 작성자용)

### 스킬 내부에서 모드 판단

모든 VibeSpec 스킬의 Step 0 (또는 Phase 0)에 다음 로직을 추가:

```
### Step 0: 모드 판정
$ARGUMENTS에 `--interactive` 플래그가 포함되어 있는지 확인하세요.
- 포함됨 → **interactive 모드**: 모든 AskUserQuestion 체크포인트 활성화
- 미포함 → **자동 모드** (기본): 아래 "자동 모드 기본값 테이블"에 따라 기본값 자동 선택
```

### 체이닝 컨텍스트 전달

상위 스킬이 하위 스킬을 호출할 때:

```
# vs-next에서 vs-commit 호출 시
Skill 도구로 vs-commit 호출, args: "--from-chain --skip-phase7"

# vs-commit은 인자에서 --skip-phase7을 보면 Phase 7을 건너뜀
```

### 안전장치 예외 처리

안전장치 카테고리에 해당하는 체크포인트는 다음과 같이 명시:

```
> **안전장치**: 이 체크포인트는 --interactive 플래그와 무관하게 항상 사용자 확인을 받습니다.
>   사유: {파괴적 명령 / 강제 완료 / 머지 충돌 / 백업 덮어쓰기 / secrets 감지}
```

## 측정 기준

**Before/After 비교 지표**:
- 스킬별 `AskUserQuestion` 호출 분기점 수
- 통상 세션(`/vs-plan → /vs-next×5 → /vs-commit → /vs-plan-verify`)에서의 총 질문 수

**목표**: 통상 세션에서 총 질문 수 **50%+ 감소**.
