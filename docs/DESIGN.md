## Design Philosophy

VibeSpec은 CLI 전용 도구로, 터미널 환경에서의 가독성과 일관성을 최우선합니다.

- **터미널 네이티브**: 웹 UI 없이 순수 터미널 출력으로 모든 정보를 전달
- **유니코드 활용**: 박스 드로잉, 트리 커넥터, 블록 문자로 구조적 시각화
- **점진적 색상 도입**: picocolors 기반 시맨틱 컬러로 상태 구분 강화
- **반응형 폭**: 터미널 너비에 따라 출력을 동적 조절

<!-- [REQUIRED] Color System -->
## Color System

### 시맨틱 컬러 (picocolors 기반, 도입 예정)

| 역할 | 색상 | 용도 | 예시 |
|------|------|------|------|
| Success | `green` | 완료, 통과 | 태스크 완료, 테스트 통과 |
| Error | `red` | 실패, 에러 | 빌드 실패, 검증 실패 |
| Warning | `yellow` | 경고, 주의 | 미완료 항목, 충돌 감지 |
| Info | `blue` | 정보, 안내 | 진행 상태, 힌트 |
| Muted | `gray` | 보조, 비활성 | 타임스탬프, ID, 부가 정보 |
| Default | (없음) | 일반 텍스트 | 본문, 설명 |

### 색상 정책

- `--no-color` 플래그 또는 `NO_COLOR` 환경변수 시 색상 비활성화
- CI 환경 자동 감지 (`CI=true` 시 색상 비활성화 권장)
- 색상은 항상 시맨틱 역할 기반으로만 사용 (장식 금지)

## Typography & Icons

### 상태 아이콘 (이모지 기반)

| 상태 | 이모지 | Fallback (no-emoji) | 용도 |
|------|--------|---------------------|------|
| done | ✅ | `[x]` | 완료 |
| in_progress | 🔄 | `[>]` | 진행 중 |
| blocked | ❌ | `[!]` | 차단 |
| todo | ⬜ | `[ ]` | 대기 |
| skipped | ⏭️ | `[-]` | 건너뜀 |
| warning | ⚠️ | `[W]` | 경고 |

### 우선순위 표시

| 레벨 | 표시 | 용도 |
|------|------|------|
| critical | `!!!!` | 즉시 대응 |
| high | `!!! ` | 높은 우선순위 |
| medium | `!!  ` | 중간 우선순위 |
| low | `!   ` | 낮은 우선순위 |

### 넘버링

- 순서 목록: `①②③④⑤⑥⑦⑧⑨⑩` (원문자 숫자)

<!-- [REQUIRED] Layout System -->
## Layout System

### 출력 폭

- **기본**: `process.stdout.columns` 참조 (동적 적응)
- **최소 폭**: 40자 (미만 시 줄바꿈 허용)
- **Fallback**: 터미널 폭 감지 불가 시 80자 기본값
- **기존 하위호환**: 박스 드로잉 55자 고정 → 동적 폭으로 점진 마이그레이션

### 박스 드로잉

```
┌─────────────────────────────────────────────────────┐
│ 제목                                                 │
├─────────────────────────────────────────────────────┤
│ 내용                                                 │
└─────────────────────────────────────────────────────┘
```

- 문자: `┌`, `─`, `┐`, `│`, `├`, `┤`, `└`, `┘`
- 내부 패딩: 좌우 1칸 공백

### 트리뷰

```
├─ 부모 항목
│  ├─ 자식 항목 1
│  └─ 자식 항목 2 (마지막)
└─ 마지막 부모
```

- 커넥터: `├─` (중간), `└─` (마지막), `│` (계속)
- 들여쓰기: 레벨당 3칸

### 테이블

- 정렬: `padRight()` 기반 고정 폭 컬럼
- 구분선: 없음 (공백 정렬만 사용)
- 기본 컬럼 폭: ID(14), Title(26), Status(12)

### 구분자

- 인라인 통계: `·` (middle dot) — 예: `done 3 · active 1 · blocked 0`
- 섹션 구분: 빈 줄 1개

<!-- [REQUIRED] Progress Indicators -->
## Progress Indicators

### 진행바

```
████████░░░░░░░░░░░░ 40%
```

- 채움: `█`, 빈칸: `░`
- 기본 폭: 20칸
- 퍼센트 표시: 바 우측

### 타임라인 바

- 동일한 `█` 문자, 최대 10칸
- 상대적 크기 비교용

## Spacing & Rhythm

| 컨텍스트 | 규칙 |
|----------|------|
| 섹션 간 | 빈 줄 1개 |
| 박스 내부 패딩 | 좌우 1칸 |
| 트리 들여쓰기 | 레벨당 3칸 |
| 테이블 컬럼 간격 | padRight로 고정 폭 |
| 경고 메시지 들여쓰기 | 2칸 (`  - [type] message`) |

## Output Modes

| 모드 | 플래그 | 동작 |
|------|--------|------|
| Human | (기본) | 포맷팅된 텍스트 출력 |
| JSON | `--json` | `JSON.stringify(data, null, 2)` |
| Verbose | `--verbose` | 에러 시 스택 트레이스 포함 |
| No Color | `--no-color` | 색상 비활성화 |

<!-- [OPTIONAL] Responsive Breakpoints -->
## Responsive Breakpoints

CLI 터미널 환경의 반응형 기준:

| 폭 범위 | 분류 | 동작 |
|---------|------|------|
| < 40자 | Narrow | 테이블 → 리스트 전환, 박스 드로잉 비활성화 |
| 40-79자 | Compact | 축약 컬럼, 제목 자르기 |
| 80-119자 | Standard | 기본 레이아웃 |
| 120자+ | Wide | 추가 컬럼 표시, 풀 타이틀 |

- 감지: `process.stdout.columns` (리사이즈 이벤트 미구독, 실행 시점 1회 측정)
- Fallback: 감지 불가 시 80자 (Standard)

<!-- [OPTIONAL] Animation -->
## Animation

- **현재**: 애니메이션/스피너 미사용. 모든 출력은 동기적 텍스트.
- **진행 표시**: 정적 진행바 (`████░░░░`) — 실시간 업데이트 없음
- **설계 원칙**: Claude Code 플러그인 특성상 stdout을 Claude가 읽으므로, 터미널 애니메이션(커서 이동, 클리어)은 의도적으로 사용하지 않음
- **향후**: 필요 시 단순 `...` 텍스트 기반 진행 표시만 허용

<!-- [OPTIONAL] Dark Mode -->
## Dark Mode

- **현재**: 터미널 기본 색상에 위임 (별도 다크/라이트 모드 없음)
- **원칙**: 시맨틱 컬러(red, green, yellow 등)만 사용하여 터미널 테마에 자동 적응
- **금지**: 하드코딩된 RGB/HEX 색상, 배경색 지정
- **접근성**: `NO_COLOR` 환경변수 지원으로 색상 완전 비활성화 가능
