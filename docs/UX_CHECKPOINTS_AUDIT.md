## 안전장치 — 자동 진행 예외 (유지)

절대 자동 진행하지 않을 5개 카테고리:

| 카테고리 | 적용 스킬 | 질문 유형 |
|---|---|---|
| **파괴적 명령** | vs-merge, vs-worktree | force push/삭제/force merge 확인 |
| **강제 완료 사유** | vs-next, vs-pick, vs-plan-verify | FAIL override 사유 입력 |
| **머지 충돌** | vs-merge | add/add 충돌 해결 선택 |
| **백업 덮어쓰기** | vs-skeleton-init, vs-design-init | 기존 .md 덮어쓰기 확인 |
| **secrets 감지** | vs-commit | .env/credentials 의심 파일 경고 |

### 통상 세션 시뮬레이션

표준 시나리오: `/vs-plan → /vs-next × 5 → /vs-commit → /vs-plan-verify`

| 모드 | 계산식 | 총 호출 수 |
|---|---|---:|
| **Before (현행)** | 4 + 6×5 + 4 + 3 | **41** |
| **After — 자동 모드 (기본)** | 1 + 2×5 + 0 + 1 | **12** |
| **After — interactive 모드** | 4 + 6×5 + 4 + 3 | **41** |

**자동 모드 감축률**: `(41 - 12) / 41 = 70.7%` → **목표 50%+ 달성** ✓

### 안전장치 5종 작동 체크리스트

자동 모드에서도 유지되어야 할 안전장치:

| # | 카테고리 | 스킬 | 적용 위치 | 작동 확인 |
|---|---|---|---|:---:|
| 1 | 파괴적 명령 | vs-merge | force merge/push 체크포인트 | ✓ 건드리지 않음 |
| 2 | 강제 완료 사유 | vs-next, vs-plan-verify | FAIL override 사유 입력 | ✓ Step 0 안전장치 예외 명시 |
| 3 | 머지 충돌 | vs-merge | add/add 충돌 해결 선택 | ✓ 건드리지 않음 |
| 4 | 백업 덮어쓰기 | vs-skeleton-init, vs-design-init | 기존 .md 덮어쓰기 확인 | ✓ 건드리지 않음 |
| 5 | Secrets 감지 | vs-commit | .env/credentials 경고 | ✓ Step 0 안전장치 예외 명시 |

### `--interactive` 플래그 복원 테스트

구현된 6개 타겟 스킬 모두 Step 0 "모드 판정" 섹션이 추가되었으며, `--interactive` 플래그가 전달되면 기존 AskUserQuestion 호출 경로가 그대로 유지됩니다.

**테스트 검증**:
- [x] vs-next: `--interactive` 시 stash/워크트리/Complex/구현방식/다음작업 체크포인트 모두 활성화
- [x] vs-commit: `--interactive` 시 Phase 0/Phase 4 체크포인트 활성화
- [x] vs-plan-verify: `--interactive` 시 PASS/WARN 후속 선택 활성화
- [x] vs-plan: `--interactive` 시 백로그 중복·워크트리 추천 체크포인트 활성화
- [x] vs-exec: `--interactive` 시 Phase 2/Phase 4 체크포인트 활성화
- [x] vs-pick: `--interactive` 시 Complex 재추천 체크포인트 활성화

### 체이닝 스킵 메커니즘

- `/vs-next` 완료 후 `/vs-commit` 호출 시 `--from-chain` 또는 `--skip-phase7` 전달 → vs-commit Phase 7 완전 스킵
- 상위 스킬의 결정을 하위 스킬이 재질문하지 않음

## 결론

- **자동 모드 기본화**: 통상 세션에서 AskUserQuestion 호출 수 **41 → 12회 (-70.7%)**
- **목표 달성**: 50%+ 감소 목표 초과 달성
- **안전장치 5종**: 모두 유지, 모드 무관 작동
- **Full 복원 가능**: `--interactive` 플래그로 기존 전체 대화형 동작 100% 복원
- **체이닝 지원**: vs-next → vs-commit 등의 스킬 체이닝 시 중복 질문 제거

사용자 피드백이 있으면 기본값 테이블(`docs/UX_DEFAULTS.md`)을 조정할 수 있습니다.
