# VibeSpec Plugin - 사용 가이드

> **Spec-Driven Development (SDD)** 기반의 Claude Code 플러그인
> "Spec first, Code second" — 스펙을 먼저 작성하고, 코드를 나중에 작성합니다.

## 3. 전체 명령어 맵 (41개 스킬)

### 3-1. 핵심 SDD 명령어

```
┌─────────────────────────────────────────────────────────────┐
│                    핵심 SDD 워크플로우                        │
├──────────────┬──────────────────────────────────────────────┤
│  /vs-setup   │ 초기 설정 & SDD 가이드                       │
│  /vs-plan    │ 스펙 작성 → 플랜 생성 → 태스크 분해          │
│  /vs-next    │ 다음 pending 태스크 가져와 실행               │
│  /vs-pick    │ 특정 태스크를 선택하여 실행                   │
│  /vs-exec    │ 플랜의 모든 태스크를 배치 실행                │
│  /vs-dashboard│ 진행률, 알림, 속도, ETA 조회                │
│  /vs-review  │ 스펙/태스크를 인터랙티브하게 편집             │
│  /vs-commit  │ 태스크 ID 연동 커밋                           │
│  /vs-next    │ 다음 태스크 (세션 복원 포함)                   │
└──────────────┴──────────────────────────────────────────────┘
```

  │  freeze:  Edit outside scope ──▶ ❌ BLOCKED│
  │  normal:  Edit in scope ──▶  ✅ ALLOWED    │

## 5. 실전 시나리오별 사용법

### 시나리오 A: 새 기능 개발 (Full SDD)

```
Step 1: 아이디어 구체화
  $ /vs-ideate
  "사용자 프로필 페이지를 만들고 싶어"
  → 6가지 질문으로 구조화 → Go/No-Go 판단

Step 2: 플랜 생성
  $ /vs-plan
  → SDD 스펙 작성 (Overview → Requirements → Data Model → API → Edge Cases)
  → 15-30분 단위 태스크로 자동 분해
  → 각 태스크에 Acceptance Criteria 포함

Step 3: 워크트리 생성 (선택)
  $ /vs-worktree
  → 격리된 브랜치에서 작업
  → 의존성 자동 설치
  → 테스트 베이스라인 확인

Step 4: 태스크 실행
  $ /vs-next        (하나씩 순차 실행)
  또는
  $ /vs-exec        (전체 배치 실행)
  → TDD: RED → GREEN → REFACTOR
  → 각 태스크 완료 시 자동 검증

Step 5: 진행 확인
  $ /vs-dashboard
  → 진행률 바, 블로킹 태스크, ETA 표시

Step 6: 코드 리뷰 & QA
  $ /vs-code-review  (버그 탐지)
  $ /vs-qa           (6-에이전트 QA)
  $ /vs-security     (보안 스캔)

Step 7: 머지 & 릴리즈
  $ /vs-merge        (squash-merge)
  $ /vs-release      (버전 범프 + GitHub Release)
```

### 시나리오 B: 간단한 버그 수정

```
  $ /adhoc
  "로그인 버튼이 모바일에서 안 보여요"
  → 스펙/플랜 없이 즉시 수정
  → 검증 후 커밋
```

### 시나리오 C: 이전 세션 이어하기

```
  $ /vs-next
  → 마지막 stash 자동 감지 및 복원
  → 다음 태스크 시작
  → 다음 태스크 제안

  $ /vs-dashboard
  → 현재 진행률 확인

  $ /vs-next
  → 다음 태스크 실행
```

### 시나리오 D: QA 전담 세션

```
  $ /vs-qa              (전체 QA 실행)
  $ /vs-qa-status       (결과 조회)
  $ /vs-qa-findings     (이슈 관리)
  $ /vs-acceptance      (AC 기반 검증)
  $ /vs-browse          (탐색적 QA)
```
